# Obsidian Document Generator

현재 대화에서 핵심 내용(계획, 분석, 결정사항, 작업 결과 등)을 추출하여 옵시디언 마크다운 문서로 생성합니다.

## 트리거

다음 문구가 포함되면 이 스킬을 자동 호출:
- "문서로 만들어줘"
- "문서 만들어줘"
- "옵시디언에 저장"
- "옵시디언에 문서로"
- "문서화해줘"

## 저장 경로

기본 경로: `/Users/aron/Library/Mobile Documents/iCloud~md~obsidian/Documents`

### 문서 유형별 하위 경로

| 유형 | 경로 | 설명 |
|------|------|------|
| 개발 계획 | `01. Projects/` | ralplan, 설계, 아키텍처 계획 |
| 기술 문서 | `01. Docs/` | API, 기술 레퍼런스, 가이드 |
| 회의/분석 | `00. Inbox/` | 분류 불명확한 문서 |

## 파일명 규칙

`{YYYY-MM-DD}_{주제 요약}.md`

예시:
- `2026-03-30_ERP_인증모듈_리팩토링_계획.md`
- `2026-03-30_API_연동_가이드.md`

## 문서 포맷

```markdown
---
created: {YYYY-MM-DD}
type: {plan|doc|analysis|reference}
project: {프로젝트명 또는 없음}
tags: [관련 태그]
---

# {문서 제목}

## 배경
{왜 이 작업/계획이 필요한지}

## 핵심 내용
{대화에서 추출한 주요 내용}

## 결정사항
{합의된 사항, 선택한 방향}

## 다음 단계
{후속 작업이 있다면}
```

## 실행 절차

1. 대화 컨텍스트에서 핵심 내용을 파악
2. 문서 유형을 판별하여 적절한 하위 경로 선택
3. 마크다운 frontmatter + 본문 구성
4. Write 도구로 파일 생성
5. 생성된 파일 경로를 사용자에게 알려줌

## 주의사항

- 코드 블록, 테이블 등 마크다운 서식을 적극 활용
- 대화 내용을 그대로 복사하지 않고, 핵심만 정리
- 민감 정보(비밀번호, API 키 등)는 제외
- 사용자가 경로나 파일명을 지정하면 그것을 우선 사용
